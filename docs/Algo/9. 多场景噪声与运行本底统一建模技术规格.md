## 1. 文档目的

本规格定义一套统一建模框架，把被监测系统中的干扰分成两大类并统一管理：

- **外部背景**：设备停机或运行时，由道路车辆、邻近设备、地面传递、工频耦合、结构共振等带来的环境扰动。
    
- **运行本底**：设备正常工作时，由机构运动、工艺过程、物料扰动、流体扰动、冲击循环、变频控制等自然产生的“正常但复杂”的振动成分。
    

本规格服务于四个目标：

1. 三态判断中的阈值与置信度修正。
    
2. 稳定态下的近似转速推断有效性判断。
    
3. 故障诊断前的背景抑制与本底分离。
    
4. 多设备、多工况、多场景现场的长期自适应。
    

系统默认使用 **LIS2DH12TR** 作为低功耗前端，利用其两套独立中断生成器、sleep-to-wake / return-to-sleep 和 1 Hz 到 5 kHz 级输出数据率做常驻活动监测；使用 **IIS3DWBTR** 作为振动主采样前端，利用其最高 26.7 kHz ODR、约 6 kHz 平坦带宽、低噪声、FIFO 和活动/静止相关功能做宽带振动分析。IIS3DWB 的完整高性能路径以 SPI 为主。

## 2. 基本思想

统一模型不再把“噪声”视为单一来源，而是把观测信号分解为：

$x(t)=b_{ext}(t|c_{env})+b_{op}(t|m,n,L,\phi)+b_{ctrl}(t)+r_{fault}(t)+e(t)$

其中：

- $b_{ext}(t|c_{env})$：外部背景，受环境场景 $c_{env}$ 影响。
    
- $b_{op}(t|m,n,L,\phi)$：运行本底，受设备模式 $m$、速度 $n$、负载 $L$、循环/相位 $\phi$ 影响。
    
- $b_{ctrl}(t)$：启停、变频、调速、执行机构切换带来的控制过渡成分。
    
- $r_{fault}(t)$：希望用于诊断的异常残差。
    
- $e(t)$：传感器与量化噪声。
    

这意味着：**OFF 模型只描述“设备未运行时的外部背景”，不能替代“设备运行时的正常本底模型”。** 对变速旋转机械，NI 的资料明确指出，频域 FFT 在转速变化时会把相关成分拉宽、重叠，而 order analysis 更适合分析随转速变化的旋转/往复式设备；同时，NI 也提供了不依赖转速传感器、从振动信号生成 speed profile 的 tachless 工具，但这仍然属于“由振动估计速度”，不是天然真值。

## 3. 适用范围

本规格适用于以下设备与现场：

- 定速电机、VFD 电机、风机、泵、压缩机、搅拌机。
    
- 冲压机、往复式、周期冲击设备。
    
- 机房靠近道路、邻机扰动明显、地基传递复杂的场景。
    
- 无专业转速传感器，或转速信号不可靠的场景。
    

本规格不假定现场是平稳背景。相反，它默认外部背景和运行本底都可能是**多模态、非平稳、随场景切换**的。

## 4. 总体架构

统一建模由三层组成：

### 4.1 状态层

由上层状态机给出：

- `OFF`
    
- `TRANSIENT`
    
- `STABLE`
    

规则要求：

- `OFF` 允许更新外部背景模型。
    
- `STABLE` 允许更新运行本底模型。
    
- `TRANSIENT` 不允许更新任一主模型。
    

### 4.2 场景层

在 `OFF` 或 `STABLE` 前提下，再判断当前属于哪个背景/工况场景：

- 外部场景：$c_{env}$
    
- 运行工况：$(m,n,L,\phi)$
    

### 4.3 模型层

维护两个模型库：

- **外部背景模型库**：$\mathcal{B}_{ext}={B_1,B_2,\dots,B_K}$
    
- **运行本底模型库**：$\mathcal{B}_{op}={O_{m,n,L,\phi}}$
    

最终推理时由选择器给出当前有效模型：

$\hat B_{ext}(t)=\sum_k p(c_{env}=k|x),B_k$

$\hat B_{op}(t)=\sum_j p(j|x),O_j$

场景不确定时，应采用保守上包络或高分位融合，而不是激进地选单一模型。

## 5. 外部背景模型规范

### 5.1 定义

外部背景模型描述“设备自己不运行时”或“设备运行但外部仍存在”的环境扰动。

### 5.2 典型场景

推荐最少支持以下场景：

- `Quiet_OFF`
    
- `Rumble_OFF`
    
- `Passby_Light_OFF`
    
- `Passby_Heavy_OFF`
    
- `Impulsive_OFF`
    
- `Unknown_OFF`
    

### 5.3 模型内容

每个外部背景模型 $B_k$ 至少包含：

- 慢变偏置：$\mu_k,\Sigma_k$
    
- 宽带 PSD 基线：$S_{k,50}(f),S_{k,95}(f)$
    
- 固定干扰线表：$L_k$
    
- 脉冲扰动统计：$I_k$
    

这里主模型必须建在 **PSD 域**。原因是噪声底会随 FFT bin 宽度变化，而 PSD 归一化到单位带宽后更适合作为可重复的噪声底参考。NI 对噪声底与 PSD 的说明就是基于这个原则。

### 5.4 建模信号

- LIS2DH12TR：用于低功耗活动度与粗场景分层。
    
- IIS3DWBTR：用于宽带 PSD、线谱、脉冲统计建模。
    

IIS3DWB 的高带宽、低噪声、工业振动定位决定了它应作为外部背景频域模型的主数据源。

## 6. 运行本底模型规范

### 6.1 定义

运行本底模型描述“设备在正常工作时，本来就应存在”的振动成分。它不是纯噪声，而是**正常工况基线**。

### 6.2 模型分解

运行本底定义为：

$b_{op}(t|m,n,L,\phi)=d_{sync}(t|m,n,L,\phi)+u_{rand}(t|m,n,L,\phi)$

其中：

- $d_{sync}$：同步/周期性/阶次相关成分。
    
- $u_{rand}$：该工况下的随机本底。
    

### 6.3 适用对象

- 对旋转设备，重点按 **模式 + 速度 + 负载** 建模。
    
- 对周期冲击设备，重点按 **循环相位** 建模。
    

NI 的资料把 order analysis 定义为适合分析转速随时间变化的旋转或往复式机械的方法，且“第一阶就是转速，n 阶是 n 倍转速”；同时，vector averaging / coherent averaging / time or angle synchronous averaging 能在 even-angle 信号上降低噪声底、保留同步成分。

### 6.4 运行本底模型内容

每个运行本底模型 $O_{m,n,L,\phi}$ 至少包含：

- 同步模板：$T_{sync}$
    
- 条件 PSD：$S_{op,50}(f),S_{op,95}(f)$
    
- 能量分布：$E_{50},E_{95}$
    
- 关键阶次/主频参考：$F_{ref}$ 或 order template
    
- 冲击统计：$C_{peak},C_{dur}$
    
- 置信度边界：$Q_{stable}$
    

### 6.5 设备类型特化

- **搅拌机/流体设备**：重点建模速度带、负载带、物料状态相关随机本底。
    
- **冲压机/往复式设备**：重点建模循环相位模板，而不是只建单一频谱模板。
    
- **VFD 电机**：重点建模速度相关本底，而不是单一固定频率模板。
    

## 7. 场景与工况上下文定义

### 7.1 外部上下文

外部上下文 $c_{env}$ 由以下特征推断：

- LIS 活动等级占空比
    
- IIS 低频带能量
    
- 宽带 PSD 形状
    
- 线谱数量与稳定性
    
- 冲击事件率
    
- 持续时间与包络形状
    

### 7.2 运行上下文

运行上下文 $(m,n,L,\phi)$ 由以下信息推断：

- 当前状态：必须为 `STABLE`
    
- 近似速度：$\hat n$
    
- 负载代理量：能量、带能量比、工艺侧先验
    
- 相位/循环位置：若设备具周期结构
    
- 外部先验：用户提供的额定转速、工艺阶段、模式编号
    

无独立转速传感器时，近似速度只允许在 `STABLE` 前提下作为诊断级输入；在 `TRANSIENT` 中最多只能作为内部候选轨迹。NI 的 tachless speed profile 工具说明了可以从振动生成速度轮廓，而 order analysis 需要把振动与速度变化结合起来理解。

## 8. 特征规范

### 8.1 外部背景特征

建议至少提取：

- 低速活动度：$A_L(k)$
    
- 宽带 RMS：$E_H(k)$
    
- 低频能量比：$R_{LF}(k)$
    
- 谱斜率：$\beta(k)$
    
- 线谱强度表：$L(k)$
    
- 冲击率：$I(k)$
    

### 8.2 运行本底特征

建议至少提取：

- 主频或主阶次：$F_H(k)$ / order peaks
    
- 同步模板相似度：$C_{sync}(k)$
    
- 条件 PSD 偏离：$D_{psd}(k)$
    
- 带内净能量：$E_{net}(B)$
    
- 冲击相位一致性：$C_{\phi}(k)$
    
- 窗口间稳定性：$R_E(k),R_F(k)$
    

### 8.3 PSD 计算要求

噪声与本底的统计谱应采用 Welch PSD 或等价平均 PSD 方法，默认优先保存中位数与高分位曲线，而不是只存均值。PSD 之所以优先于普通功率谱，是因为噪声底与测量带宽的关系更可控。

## 9. 模型选择与融合

### 9.1 选择规则

当 `state = OFF` 时，选择外部背景模型：

$\hat S_{ext}(f|x)=\sum_k p_k S_{k,95}(f)$

当 `state = STABLE` 时，选择运行本底模型：

$\hat S_{op}(f|x)=\sum_j q_j S_{j,95}(f)$

其中 $p_k$ 与 $q_j$ 为场景/工况后验概率。

### 9.2 保守策略

若后验分布不尖锐，应使用：

- 高分位融合，或
    
- 上包络融合
    
不得在高不确定场景下强行选择单一模型。

### 9.3 运行前背景继承

设备从 `OFF -> TRANSIENT -> STABLE` 的一次完整启动中，应优先继承启动前最近背景场景作为当前外部背景初值。

## 10. 学习与更新规则

### 10.1 外部背景模型更新

仅允许在以下条件下更新：

- `state = OFF`
    
- `confidence >= C_{off}`
    
- 窗口通过洁净性筛选
    
- 无温度、供电、安装状态突变
    

更新公式可采用：

$\theta^{new}=(1-\alpha)\theta^{old}+\alpha \hat\theta$

其中 $\alpha$ 应较小。

### 10.2 运行本底模型更新

仅允许在以下条件下更新：

- `state = STABLE`
    
- `speed_valid = true` 或至少 `tentative` 且置信度高
    
- 工况标签稳定
    
- 未被故障检测模块标记为异常
    

### 10.3 禁止更新条件

以下状态不得更新主模型：

- `TRANSIENT`
    
- 置信度不足
    
- 发生脉冲污染
    
- 场景未知且不确定度过高
    
- 已知异常运行
    

## 11. 模型在诊断链路中的使用

### 11.1 外部背景修正

运行态某频点显著性定义为：

$G_{ext}(f)=10\log_{10}\frac{S_{run}(f)}{\max(\hat S_{ext}(f),\epsilon)}$

该量用于判断该成分是否显著高于环境背景。

### 11.2 运行本底修正

对已经高于外部背景的频点，再判断其是否偏离正常运行本底：

$G_{op}(f)=10\log_{10}\frac{S_{run}(f)}{\max(\hat S_{op}(f),\epsilon)}$

### 11.3 诊断残差

推荐的残差构造是：

$r_{diag}=x-\hat b_{ext}-\hat d_{sync}$

其中 $\hat d_{sync}$ 可由同步模板、阶次模板或相位模板得到。  
对同步成分的提取，可利用 coherent/vector averaging 或 time/angle synchronous averaging 的思想，在 even-angle 或同步条件下保留同步分量、降低非同步噪声底。

### 11.4 禁止事项

不得默认做以下处理：

- 直接用 OFF 原始波形减去运行波形。
    
- 直接把运行本底全部当成“应消除的噪声”。
    
- 在 `TRANSIENT` 里估计出的速度直接作为诊断真值。
    

## 12. 与三态判断的接口

统一建模系统必须与三态状态机紧耦合：

- `OFF`：允许外部背景建模。
    
- `TRANSIENT`：仅做缓存与候选分类，不做主模型学习。
    
- `STABLE`：允许运行本底建模和近似速度进入诊断链路。
    

这意味着三态判断不仅是状态输出，也是模型学习边界控制器。

## 15. 验收要求

至少验证以下能力：

1. 路边机房、邻机扰动等多场景下，外部背景模型可区分并稳定收敛。
    
2. 搅拌机、冲压机等设备的正常运行本底不会被误判为故障。
    
3. 使用统一模型后，三态误报率下降。
    
4. 使用统一模型后，稳定态近似转速有效性提高。
    
5. 场景不确定时，系统表现为“更保守”，而不是“更激进”。
    

## 16. 核心结论

这版规格的核心不是“把噪声做得更复杂”，而是把问题分清楚：

- **外部背景** 解决的是“外面太吵”。
    
- **运行本底** 解决的是“设备自己正常工作也很吵”。
    
- **三态判断** 解决的是“什么时候可以学模型、什么时候只能观察”。
    
- **转速有效性** 解决的是“什么时候才允许把振动主频解释成诊断级速度”。
