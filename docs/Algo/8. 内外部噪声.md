你这个补充把模型又推进了一层，而且是非常关键的一层：**不仅有外部背景噪声，还有设备自身在正常工作时产生的“工艺/机构本底”**。  
我先把这两类成分在信号处理上的边界理顺一下，再给你一个统一的分层建模框架，不然“噪声”这个词会把正常工况振动和故障特征混在一起。

Thought for 1m 25s

对，这一类必须纳入，而且它和“外部停机噪声”不是同一个层级。

更准确地说，前面讨论的主要是 **外部背景噪声**；你现在补充的是 **设备在正常工作时由工艺、机构、载荷变化自身产生的运行本底**。对故障诊断来说，后者往往更棘手，因为它和真正的故障特征都发生在“设备已运行”的条件下。对非平稳、变工况机械，文献本身就把这类运行中的非平稳/循环平稳振动视为状态监测的核心难点，传统单一静态模型很容易失配。

所以我们要把原来的“停机噪声模型”升级成一个统一框架。建议把观测信号写成：

$x(t)=b_{ext}(t|c_{env})+b_{op}(t|m,n,L,\phi)+b_{ctrl}(t)+r_{fault}(t)+e(t)$

这里：

- $b_{ext}(t|c_{env})$：外部背景噪声，按环境场景变化
    
- $b_{op}(t|m,n,L,\phi)$：设备正常运行本底，受模式 $m$、速度 $n$、负载 $L$、工艺相位 $\phi$ 影响
    
- $b_{ctrl}(t)$：启停、变频、执行机构动作带来的控制过渡成分
    
- $r_{fault}(t)$：真正希望提取的故障残差
    
- $e(t)$：传感器与量化误差
    

这样一来，之前的 OFF 噪声模型只是其中一层，而不是全部。

这里最关键的概念区别是：  
**搅拌机、冲压机这类设备在运行时产生的很多“噪声”，其实不能简单视为噪声，更应视为“正常运行本底”或“工艺本底”。**  
它们不是要被无脑消掉，而是要先建模，再从中分离出异常偏离。

对旋转类设备，NI 明确指出，很多振动/噪声成分与转速成比例，因此更适合用 order analysis 来看；而 time synchronous averaging（TSA）能够提取与轴相关的周期成分，并抑制随机噪声和其他无关扰动。

这意味着，对搅拌机、风机、泵、VFD 电机一类设备，运行本底至少要拆成两部分：

第一部分是 **确定性本底**，例如与转速同步的 1X、2X、叶频、齿频、周期性调制。  
第二部分是 **随机工艺本底**，例如流体扰动、物料不均匀、混合过程随机冲击、载荷微波动。

从模型上可以写成：

$b_{op}(t|m,n,L,\phi)=d_{sync}(t|m,n,L,\phi)+u_{rand}(t|m,n,L,\phi)$

其中 $d_{sync}$ 是同步/周期性成分，$u_{rand}$ 是条件化随机背景。

对搅拌机，我建议把“正常运行本底”按 **速度带 + 负载带 + 工艺阶段** 建模，而不是只按“开机”建一个模板。也就是说，不是一个 `Mixer_ON_Model`，而是一组：

$N_{mix}^{(m,n,L)}$

每个模型里同时保存：

- 同步阶次模板
    
- 该工况下的 PSD 基线
    
- 该工况下的方差/分位数界限
    
- 必要时再加包络或冲击统计
    

这样后续故障判断看的是“当前特征是否偏离同工况下的正常本底”，而不是偏离一个全局平均。

冲压机、往复式、周期冲击类设备更进一步。  
这类设备的“正常工作振动”本身就带有强烈的周期冲击和相位结构，广义宽带波形和普通频谱往往会把正常冲击和故障冲击混在一起。相关工程资料明确指出，对往复式机械，常规宽带波形/频谱往往价值有限，正常振动会掩盖故障信息，更合适的做法是按曲轴位置或循环位置进行条件化分析。

所以对冲压机，我不建议只建“运行 PSD 模型”，而应该建 **循环相位模型**：

$N_{press}^{(\phi)}$

也就是把一个工作循环拆成若干相位段，例如：

- 接近段
    
- 接触/冲击段
    
- 保压/成形段
    
- 回程段
    
- 空载等待段
    

每个相位段分别维护：

- 相位同步平均模板
    
- 相位条件 PSD
    
- 相位条件方差
    
- 冲击峰值分布
    

从方法上，这更接近 **cycle-synchronous averaging / cyclostationary 建模**，而不是单一稳态噪声底。

所以现在整套体系应当从“多场景 OFF 噪声模型”升级成两层模型库：

第一层：**外部背景模型库**  
$ {N_{off}^{(1)},N_{off}^{(2)},...,N_{off}^{(K)}} $

第二层：**运行本底模型库**  
$ {N_{on}^{(m,n,L,\phi)}} $

前者解决“机房在路边、重车经过、邻机干扰”这类外部环境问题；后者解决“设备自己正常工作就很吵、很冲击、很宽带”这类内部工艺问题。

真正用于故障诊断时，不应该问“当前谱比 OFF 大多少”，而应该问两件事：

1. 它是否显著高于当前环境背景
    
2. 它是否显著偏离当前工况下的正常运行本底
    

所以诊断残差更合理的写法是：

$r_{diag} = x - \hat b_{ext} - \hat d_{sync}$

再去看剩余部分是否还显著高于对应工况下的随机本底。  
这里的 $\hat d_{sync}$ 对旋转机械通常可由 TSA、同步平均、阶次模板得到；对冲压机这类周期设备，则更适合用循环同步平均或相位模板。TSA 用来保留与参考同步的成分并平均掉其他不相关部分，这正适合把“正常周期机构动作”先提出来。

这也会反过来影响你前面的“三态判断”：

- **OFF**：用外部背景模型
    
- **STABLE**：不是只有“设备稳定”，还意味着“已进入某个可建模的运行本底工况”
    
- **TRANSIENT**：既不能更新 OFF 模型，也不能更新 ON 本底模型
    

所以从软件架构上，建议把模型管理器正式拆成三部分：

- `ExternalNoiseModelBank`：外部 OFF 场景库
    
- `OperationalBaselineBank`：设备正常运行本底库
    
- `ContextSelector`：根据状态、速度、负载、相位选择当前应使用的模型
    

最终结论就是这句：

**我们刚讨论的话题，不能再只叫“停机噪声模型”，而应该扩展为“外部背景 + 运行本底”的统一干扰建模框架。**
