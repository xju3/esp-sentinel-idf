**IIS3DWBTR 可以作为“无转速传感器”的在线转速估计器，但它不能在没有外部真值的情况下完成“标定”。**  
如果你后续所有分析都要建立在“采样期间的实际转速”上，那么工程上必须把两件事分开：
1. **转速估计**：用 MEMS 振动信号在线反演转速。
2. **转速标定/确认误差界**：至少做一次与光电转速计或编码器的同步实验，建立误差上限、失效场景和置信度门限。
这不是抬杠，而是计量学底线。没有独立参考，你最多能说“估计到了转速”，不能说“标定了转速”。

## 1. 问题定义

目标不是“看起来差不多”的转速，而是**足以支撑后续所有频域/阶次分析的实际转速**。这意味着算法必须满足三个要求：
- **可识别**：能在 10–100 Hz 内找到与真实轴频对应的 1X 成分，而不是把 2X、3X、工频、齿频或结构共振误判为转速。
- **可量化误差**：必须给出误差预算，而不是仅给出一个估计值。
- **可置信度判定**：当信号不满足条件时，算法应输出“不可信”，而不是硬给一个错误转速。

从这个定义出发，**“直接做一次 FFT，然后取 10–100 Hz 内最大峰”不够严谨**。它只能作为初值，不应作为最终转速真值来源。

## 2. IIS3DWBTR 的硬件可行性与边界

IIS3DWBTR 的核心优势，是它并不是普通低速姿态类 MEMS，而是面向工业振动的宽带低噪声器件。ST 的规格书给出：频率响应 **dc–6 kHz（±3 dB）**，输出数据率 **26.667 kHz**，支持时间戳，且可通过中断间隔或寄存器信息获得实际 ODR。器件还明确提示：**完整特性应使用 SPI，I²C 只支持受限模式且不推荐。**

但是这个器件有两个与你问题直接相关的边界：

**边界一：不要启用高通路径。**  
应用笔记给出的高通/斜率滤波最低截止就是 **ODR/800**。按 26.667 kHz 计算，约为 **33.3 Hz**。这意味着一旦走高通链路，**600–2000 rpm（10–33.3 Hz）** 的 1X 成分会被明显削弱甚至被“设计性过滤掉”。所以，转速估计必须使用低通链路，且尽量避免任何会动到 10–100 Hz 的 AC 耦合。

**边界二：低频可测，不等于低频已被充分规格保证。**  
数据手册虽然给出 **dc–6 kHz** 的平坦响应，但噪声密度的规格注释写得很清楚：该噪声指标对应的频段是 **100 Hz–6.3 kHz**。这并不表示 10–100 Hz 不能测，而是表示 **10–100 Hz 特别是 10–20 Hz** 这段，你不能仅靠 datasheet 上的噪声数字来承诺误差，必须做实机验证。

## 3. 信号模型与“为什么 1X 不是理所当然可见”

对于装在机壳上的加速度计，理想化信号可以写成：

$$x(t)=\sum_{m=1}^{M} A_m \cos\!\big(2\pi m f_r t+\phi_m\big)+n(t)+r(t)$$

其中 $f_r$​ 是转频，$$m=1,2,3,\dots$$是阶次，$$n(t)$$是噪声，$$r(t) $$包括共振、齿轮啮合、电磁激励、工频耦合等。
关键点在于：**机壳振动里的最大峰未必是 1X。**  
在很多机器上，2X、叶频、槽频、电磁力、结构共振峰都可能比 1X 更强。因此，任何把“10–100 Hz 内最高峰 = 转速”的做法，都只是**经验法**，不是**可靠法**。
所以真正严谨的判据不是“峰最大”，而是“该候选频率是否在谐波结构、相位连续性、邻窗一致性和多轴一致性上都像 1X”。

## 4. FFT 方法何时成立，何时失效

### 4.1 恒速时，FFT 是成立的

当电机在一个观测窗 TTT 内近似恒速时，FFT 的频率分辨率为$$\Delta f=\frac{1}{T}$$
如果只是取最近频点，频率量化误差大约为$\pm \frac{1}{2T}​$
对应转速误差：$\Delta n \approx \pm \frac{30}{T}\ \text{rpm}$

这就是你之前说的“在校正 ODR 的情况下，误差与采集时间成反比”的来源。这个结论在**恒速、正确锁定 1X、SNR 足够高**时是成立的。

### 4.2 但 FFT 只能解决“稳态平均转速”，不能自动保证“实际转速”

只要出现以下任一情况，单段 FFT 就不再是充分解：
- 采样窗内有明显加减速；
- 1X 不是最强峰；
- 10–100 Hz 内存在多个相近强峰；
- 低频 SNR 不足；
- 结构共振把机壳响应放大到掩盖真实 1X；
- 安装位置导致某一轴几乎看不到 1X。

综述论文明确指出，无转速传感器速度估计并不存在“对所有工况都最优”的单一方法；在非平稳工况下，很多方法需要转到**相位解调**或**时频跟踪**框架，常见路线包括时频脊线跟踪和 **Vold–Kalman**。

因此，**如果你的后续分析“致命依赖”实际转速，那么单纯 FFT 只能做粗估，不能独任最终基准。**
