## 1. 文档目的

本规格定义在 **停机状态 OFF** 下建立噪声模型的方法，用于服务以下目标：

- 修正运行态与稳态判定阈值
    
- 为频谱特征提供噪声底参考
    
- 为近似转速推断提供有效性约束
    
- 为故障诊断提供背景噪声剔除与置信度修正依据
    

本规格不要求在 OFF 状态下保存“可逐点相减的时域模板”，而是要求建立 **统计意义上的噪声基线模型**。这一点的核心原因是：噪声测量更适合在 PSD 域表达，因为 PSD 归一化到 1 Hz 带宽后，更适合做可重复的噪声底比较；普通功率谱中的噪声水平会随频率分辨率变化。

## 2. 适用范围

本规格适用于：

- 电机、风机、泵、压缩机等旋转设备
    
- 使用 LIS2DH12TR 作为低功耗前端、IIS3DWBTR 作为振动主采样前端的系统
    
- 后续需要做稳态判定、近似转速推断或故障诊断的系统
    

本规格默认 OFF 状态已经由上层三态状态机确认，且模型更新仅在高置信 OFF 窗口中执行。LIS2DH12TR 具备两套独立中断生成器、FIFO、sleep-to-wake 和 return-to-sleep，适合长期背景监测；IIS3DWBTR 具备宽带低噪声采样能力，适合建立宽带噪声底模型。

## 3. 总体原则

停机噪声模型必须遵循以下原则：

1. **只在 OFF 状态下学习**
    
2. **只学习统计特征，不默认学习时域相减模板**
    
3. **主模型采用 PSD，而不是普通 FFT 幅度谱**
    
4. **运行态只读，不写入 OFF 噪声模型**
    
5. **偶发冲击、邻机扰动、异常窗口必须剔除**
    

这里第 3 条是强制要求：NI 的说明明确指出，噪声测量宜使用 PSD，因为它消除了噪声水平对 $\Delta f$ 的依赖，更适合作为噪声底基线。

## 4. 模型组成

停机噪声模型 `NoiseProfile` 至少包含四层。

### 4.1 慢变偏置层

用于描述各轴在停机时的均值、方差和协方差，服务于去均值、安装变化检测和 LIS 背景活动度建模。

建议保存：

- `bias_mean[3]`
    
- `bias_var[3]`
    
- `bias_cov[3x3]`
    

对每轴原始数据 $x[n]$，停机偏置定义为 $ \mu_x = E[x[n]] $，方差定义为 $ \sigma_x^2 = E[(x[n]-\mu_x)^2] $。

### 4.2 宽带随机噪声层

这是主模型，必须基于 IIS3DWBTR 的停机数据建立 PSD 基线。

建议保存：

- `psd_median[f]`
    
- `psd_p95[f]`
    

其中：

- $S_{n,50}(f)$ 表示停机噪声中位 PSD
    
- $S_{n,95}(f)$ 表示停机噪声 95 分位 PSD
    

IIS3DWBTR 的宽带、低噪声特性使其适合作为该层主数据源。

### 4.3 固定干扰线层

用于记录停机时长期存在的窄带线谱，例如工频、电源耦合、邻机结构传递线等。

建议保存每条干扰线：

- `freq`
    
- `amp_mean`
    
- `amp_var`
    
- `persistence`
    

该层用于运行态频带屏蔽和特征降权。

### 4.4 脉冲扰动层

用于记录停机态下偶发冲击或瞬态扰动的统计特征。

建议保存：

- `impulse_rate`
    
- `peak_quantiles`
    
- `duration_quantiles`
    

该层用于剔除污染窗口，避免少数冲击抬高噪声模型。

## 5. 数据采集条件

只有同时满足以下条件，OFF 窗口才允许进入噪声模型候选池：

- 上层状态机输出 `state = OFF`
    
- `confidence >= C_off_min`
    
- LIS 低速活动度持续低
    
- IIS 宽带能量持续低
    
- 未检测到稳定主频结构
    

LIS2DH12TR 的 sleep-to-wake / return-to-sleep 和独立中断机制适合做这种长期候选筛选；IIS3DWBTR 则用于最终的宽带确认。

## 6. 窗口洁净性筛选

即使状态已经是 OFF，候选窗口仍必须通过洁净性筛选。以下任一条件成立时，窗口不得写入噪声模型：

- 窗口 RMS 高于 OFF 历史上界
    
- 峰值因子异常偏高
    
- 与前后窗口相比突变过大
    
- 出现新的强窄带峰
    
- 温度、供电或安装状态刚发生变化
    

这是为了确保噪声模型描述的是“背景基线”，而不是偶发事件。

## 7. 频域建模方法

OFF 噪声主模型必须使用 **Welch PSD** 或等价的分段平均 PSD 方法。

推荐流程：

1. 对停机窗口去均值
    
2. 加 Hann 窗
    
3. 分段并计算 periodogram
    
4. 做 Welch 平均
    
5. 输出每轴或模量信号的 PSD
    

Welch 方法通过分段与平均降低谱估计方差，PSD 又适合作为噪声底表示，因此两者组合适合作为 OFF 噪声建模主方法。

推荐模量信号定义为：

$ m_H[n] = \sqrt{x_H[n]^2 + y_H[n]^2 + z_H[n]^2} $

推荐 PSD 统计量为：

$ S_{n,50}(f) = \mathrm{median}(S_i(f)) $

$ S_{n,95}(f) = q_{95}(S_i(f)) $

其中 $S_i(f)$ 为第 $i$ 个停机洁净窗口的 PSD。

## 8. 模型更新规则

噪声模型必须采用慢更新，不允许被单次窗口强烈拉动。

对任意连续统计量，建议使用指数平滑：

$ \hat X^{new} = (1-\alpha)\hat X^{old} + \alpha X^{obs} $

其中 $\alpha$ 建议取 0.01 到 0.05。

对 PSD 曲线，若实现资源允许，优先维护中位数和高分位近似值，而不是仅维护均值。这样对偶发污染更稳健。这一条是工程推断，但与 PSD 用于噪声底测量的原则一致。

## 9. 模型输出与使用方式

停机噪声模型必须至少支持以下三种使用方式。

### 9.1 频谱显著性判定

运行态某频点是否“高于背景”，应相对于 OFF 噪声底判断，而不是看绝对值。

定义显著性增益：

$G(f) = 10 \log_{10} \frac{S_{run}(f)}{\max(S_{n,95}(f), \epsilon)}$

当 $G(f)$ 超过门限时，才允许将该频点视为有效机械成分。

### 9.2 频带屏蔽与降权

若某窄带线在 OFF 模型中长期存在，则运行态中该频带应：

- 直接屏蔽，或
    
- 提高有效性门限，或
    
- 降低在故障评分中的权重
    

### 9.3 带内净能量估计

对某分析频带 $B$，可定义净能量：

$ E_{net}^2(B) = \max(E_{run}^2(B) - E_{noise}^2(B), 0) $

该操作仅适合用于 **能量级修正**，不得默认扩展为时域逐点相减。

## 10. 禁止事项

以下行为默认禁止：

1. 用 OFF 原始时域波形直接与运行态波形相减
    
2. 用 TRANSIENT 或 STABLE 窗口更新 OFF 噪声模型
    
3. 在没有洁净性筛选的情况下写入噪声模型
    
4. 仅凭单次 OFF 采样建立最终噪声模型
    

这些禁止项的核心原因是：工业停机背景通常只在统计意义上稳定，而不是逐点可重复；因此模型应服务于阈值修正、PSD 对比、SNR 计算和频带屏蔽，而不是默认做波形模板消除。这个结论是基于 PSD 适合噪声测量的原则做出的工程推断。

## 11. 软件模块要求

建议新增模块 `NoiseBaselineManager`，负责：

- OFF 洁净窗口接收
    
- 偏置层更新
    
- PSD 层更新
    
- 干扰线检测与维护
    
- 脉冲扰动统计
    
- 运行态只读接口输出
    

建议对外提供：

- `get_noise_psd()`
    
- `get_noise_floor(f)`
    
- `get_line_table()`
    
- `is_window_clean_off()`
    
- `get_noise_validity_score()`
    

## 12. 验收要求

至少验收以下项目：

- OFF 模型在不同天、不同批次下的重复性
    
- 模型更新后对稳态误报率的改善程度
    
- 模型更新后对频带误检率的改善程度
    
- 邻机扰动、工频干扰存在时的稳定性
    
- 运行态 SNR 判定的一致性
    

强制验收项：

- OFF 模型必须只由高置信 OFF 洁净窗口更新
    
- 运行态不得反向污染 OFF 模型
    
- 无 OFF 高置信数据时，模型必须保持只读旧值或回退默认值
    

## 13. 核心结论

这份规格可以压缩成四句话：

1. **停机噪声模型是基线模型，不是波形模板。**
    
2. **主模型必须建在 PSD 域。**
    
3. **模型更新只能发生在高置信、洁净的 OFF 窗口。**
    
4. **模型主要用于阈值修正、频谱显著性判断、频带屏蔽和净能量估计。**

下一步最自然的延伸，是把这份规格再细化成 **“停机噪声模型的数据结构 + 伪代码”**。
